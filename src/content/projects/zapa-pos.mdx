This platform was built to cover the day-to-day operations of a retail shoe store: a single source of truth shared across admins, cashiers, storefront personnel and the warehouse team, with every module connected in real time.

## A desktop-like interface in the browser

Navigation is **window-based**: any module can be opened in its own window, minimized, and reopened without losing state. It feels closer to a desktop OS than a typical web app, which makes it easy to work on multiple tasks at once — processing a sale while checking stock, for example. All windows stay in sync through **WebSockets**, so changes made by one user are reflected instantly for everyone else.

## Core modules

- **Inventory and catalog**: a products list with bulk import and editing, and advanced filters and sorting available directly from the column headers. A **purchases wizard** speeds up product entries during restocks.
- **Point of Sale**: sales processing with ticket printing.
- **Built-in code editor**: a fully-featured **QR, barcode and ticket editor** so label designs live inside the platform — no external tooling needed.
- **Warehouse orders**: store personnel send orders directly to the warehouse team, with end-to-end tracking.
- **Mobile-first storefront**: a touch-friendly interface for floor staff with a **camera-based scanner** and quick access to inventory stock.

## Stack

Built on **Meteor + TypeScript** (chosen for its reactive, WebSocket-first data flow), with **MongoDB** as the database, **SCSS** for styling and **Docker** for deployment.
